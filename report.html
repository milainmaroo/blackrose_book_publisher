Introduction

My name is Mi Lainmar (Norine) Oo. I claim that I built and designed this "Black Rose Publishing" website with my own creation, including Logo Design, choosing appropriate layout and making overall design decision. The website is user-friendly, simple, responsive, easy to navigate to any device, and is designed to meet the Web Standards, therefore, I believe that this website conforms to standards and satisfies the accessibility for the company's requirements. I explained more about my website and how my website would bring benefits for the company with three reasons: Technical Aspects, Aesthetics and Accessibility.

Technical Aspects

I built a design as simple as possible so that people with slow internet connections would still be esay to navigate. I attempted to meet Coding Standards when building the web pages and the website is tested to be easily viewable with any viewports and any size of devices. I applied comments on every web pages so that other web develpers can easily understand my codes and update or make changes to the website without any difficulty in the future.

Aesthetics

The creation of the website layout is inspired by books publishing websites such as Scribd , Penguin Books Australia and Allen & Unwin Australia. Choosing the right colour is fundamental for the website to build trust and satisfaction from the users or readers, therefore, I chose a combination of tawny(brownish-orange color) and gray(both dark and light) as main colours for the web pages, where tawny reflects excitement and enthusiasm and gray is a neutral colour which reflects calm and balance. I also applied Box Model, Flexbox and Grid Layouts to display and position HTML elements on my web pages. I applied favicon of my logo design in every address bar of the web browser.

Accessibility

In terms of the accessibility of the website, I used semantic HTML such as nav, section, article, footer, which makes webpages accessible for mobile devices as well as allows browsers to interpret the code better and easier so that it is also helpful for people with disabilities. Semantic HTML also makes the website’s source code easier to read for other developers and improves the website Search Engine Optimization (SEO). All the web pages are responsive, which can be accessible for all devices such as widescreen devices(desktop, laptop), meduim screen(tablet), small screen(mobile) as well as accessible for printing.
